生成日志

使用git时生成漂亮的日志

git log --oneline --decorate --color --graph

--oneline 生成文字

--color 与终端颜色相同

--graph 可视化地显示你提交的分支

git log -p

这在代码复查时很管用--在合并到你的某个分枝之前查看是其他作者提交了什么内容，或者是看看在最后发布之前所做的修改。

git log --pretty=format

可以将提交历史显示成你想要的格式。这里format的可选项包括：oneline，short，medium，full，fuller，email，raw。每种格式都有侧重的显示相关内容。

git log --before="2 weeks ago" --after="2009-01-26" --pretty=oneline

查看1月26号之后，2个星期之前的提交内容

'git log'命令后如果跟-before和-after选项，就会显示两个日期之间的提交条目。日期格式也有不同的可选格式。

漂亮的代码

 git log --graph --pretty=format:'%C(yellow)%h%Creset -%C(cyan)%d%Creset %s %Cgreen(%an, %cr)' --abbrev-commit

 git log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr)%Creset' --abbrev-commit --date=relative

自动生成log文件

 git log >log.md
