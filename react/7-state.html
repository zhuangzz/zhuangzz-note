为何要发明let和const

在老js中，也就是es5中，声明变量使用var。但是es6中推荐使用let和const。

var 的缺陷

var 总体来说，缺点：

-作用域不清晰，造成各种尴尬使用情况，具体见参考

-可以重复声明，代码有错误，不容易报错出来

对应的let和const就能避免这些问题

（TS typescript现在这么火，就是因为强类型，报错多）

let const区别

let用来声明变量。const用来声明常量，后续允许修改。const 定义一个变量后，一旦修改就会报错（当然，const 也不是绝对不能改的，详情见参考链接http://es6.ruanyifeng.com/#docs/let）。
