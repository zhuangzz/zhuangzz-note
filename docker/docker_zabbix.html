1. Document Objective

  • Launch and configure Zabbix monitoring infrastructure in container
  • Later build Grafana dashboard UI up front of Zabbix

2. Installation

2.1 Architecture

选择server-agent(active)模式进行zabbix的搭建。

为了减轻server端的压力，保持良好的用户体验，所有zabbix agent我们均使用主动(active)模式

2.2 安装说明

zabbix agent, server, proxy均采用 3.2.6 版本

2.2.1 preparations

  • docker has installed in host
  • docker image (官方下载)
    • zabbix-java-gateway download link:https://hub.docker.com/r/zabbix/zabbix-java-gateway/
    • zabbix-web-nginx-mysql download link: https://hub.docker.com/r/zabbix/zabbix-web-nginx-mysql/
    • mysql 5.7

  • docker image(build by yourself) 在启动zabbix之前，为了在zabbix server中使用curl指令，需要自行build zabbix-server-mysql image.

    • 进入zabbix docker github project https://github.com/zabbix/zabbix-docker,选择tag为3.2.6版本进行download或者clone.
    • 在该项目下，进入server-mysql/ubuntu,修改DockerFile如下:

2.3 安装流程

2.3.1 Deploy Zabbix in docker

  • Docker compose file
version: "3"

services:
  mysql-server:
    image: mysql:latest
    command: mysqld --character-set-server=utf8 --collation-server=utf8_bin
    hostname01: mysql
    environment:
    - MYSQL_DATABASE=zabbix
    - MYSQL_USER=zabbix
    - MYSQL_PASSWORD=zabbix_pass
    - MYSQL_ROOT_PASSWORD=root_pass
    volumes:
    - "/etc/timezone:/etc/timezone:ro"
    - "/etc/localtime:/etc/localtime:ro"
    deploy:
      placement:
        constraints: [node.labels.host==hostname01]
      replicas: 1

  zabbix-java-gateway:
    image: zabbix/zabbix-java-gateway:latest
    hostname01: zabbix-java-gateway
    volumes:
    - "/etc/timezone:/etc/timezone:ro"
    - "/etc/localtime:/etc/localtime:ro"
    deploy:
      placement:
        constraints: [node.labels.host==hostname01]
      replicas: 1

  zabbix-server:
    image: zabbix/zabbix-server-mysql:latest
    hostname01: zabbix-server
    environment:
    - DB_SERVER_HOST=mysql-server
    - MYSQL_DATABASE=zabbix
    - MYSQL_USER=zabbix
    - MYSQL_PASSWORD=zabbix_pass
    - MYSQL_ROOT_PASSWORD=root_pass
    - ZBX_JAVAGATEWAY=zabbix-java-gateway
    - TZ="Asia/Shanghai"
    - ZBX_TZ="Asia/Shanghai"
    - ZABBIX_TZ="Asia/Shanghai"
    volumes:
    - "/etc/timezone:/etc/timezone:ro"
    - "/etc/localtime:/etc/localtime:ro"
    depends_on:
    - mysql-server
    - zabbix-java-gateway
    ports:
    - 10051:10051
    deploy:
      placement:
        constraints: [node.labels.host==hostname01]
      replicas: 1

  zabbix-web-nginx-mysql:
    image: zabbix/zabbix-web-nginx-mysql:latest
    hostname01: zabbix-web-nginx-mysql
    environment:
    - DB_SERVER_HOST=mysql-server
    - MYSQL_DATABASE:zabbix
    - MYSQL_USER=zabbix
    - MYSQL_PASSWORD=zabbix_pass
    - MYSQL_ROOT_PASSWORD=root_pass
    volumes:
    - "/etc/timezone:/etc/timezone:ro"
    - "/etc/localtime:/etc/localtime:ro"
    depends_on:
    - mysql-server
    - zabbix-server
    ports:
    - 8085:80
    deploy:
      placement:
        constraints: [node.labels.host==hostname01]
      replicas: 1

Tips: 1. In mysql-server service, running command command: mysqld --character-set-server=utf8 --collation-server=utf8_bin 2. zabbix-web-nginx-mysql looks for zabbix-server when it's up. Defined in depends_on. So name zabbix-server instead of zabbix-server-mysql 3. Port 10051 will be used by Hyperledger. Therefore, we change it to 20051

  • Docker stack deploy
docker stack deploy -c zabbix_mysql.yaml zabbix_mysql
  • Login in web

http://10.0.0.0:8085

用户名：Admin 密码：zabbix

2.3.2 Install agent on all hosts
  • Download repo
wget http://repo.zabbix.com/zabbix/3.2/ubuntu/pool/main/z/zabbix-release/zabbix-release_3.2-1+trusty_all.deb
  • Transfer and install repo
for i in {1..4}; do scp zabbix-release_3.2-1+trusty_all.deb cm0$i:~/; ssh root@cm0$i "dpkg -i /home/ubuntu/zabbix-release_3.2-1+trusty_all.deb"; done

>

sudo apt-get purge zabbix_agent.server /etc/apt/zabbix.list

  • Update and install
for i in {1..4}; do ssh cm0$i -t "sudo apt-get update"; done
for i in {1..4}; do ssh cm0$i -t "sudo apt-get -y install zabbix-agent"; done
  • Configure firewall through ufw if it's active
sudo ufw allow 10050/tcp
  • Configure /etc/zabbix/zabbix_agentd.conf
Server=<zabbix server ip>
ServerActive=<zabbix server ip>
hostname01=<host name>
StartAgents=0 # 设置为0将完全关闭被动模式
  • Copy (append) /etc/zabbix/zabbix_agentd.conf to all monitoring targets then restart zabbix-agent.service service
for i in {1..4}; do scp zabbix_agentd.conf host0$i:/tmp/; ssh host0$i -t "sudo cat /tmp/zabbix_agentd.conf | sudo tee -a /etc/zabbix/zabbix_agentd.conf"; done
  • Specifically on zabbix-server container's host
    Specifically on the host of zabbix-server container, edit /etc/zabbix/zabbix_agentd.conf. Use container's IP instead of host IP
    ServerActive=172.19.0.6

To fix the error in /var/log/zabbix/zabbix_agentd.log on the host of zabbix-server container

3hostname0127:2hostname0171029:223856.452 failed to accept an incoming connection: connection from "172.19.0.6" rejected, allowed hosts: "172.29.167.178"

3. Zabbix 配置说明

3.1 Actions

Configure to Auto- Register the Client

  • Go to dashboard > Configuration > Actions > Select Auto registration in Event source, then client Create action

    • In Action under Actions > give a name and specify like > click Add.
    • In Operations under Actions > click New > select Add host or Add to host group in Operation type
  • Edit /etc/zabbix/zabbix-agentd.conf on each client, specify a unique value of hostname01, then restart zabbix_agentd
