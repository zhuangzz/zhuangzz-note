docker 中部署zabbix2

Zabbix Playbook

Reference

  • Zabbix in Docker

  • Zabbix agent on Ubuntu

Document Objective

  • Launch and configure Zabbix monitoring infrastructure in container
  • Later build Grafana dashboard UI up front of Zabbix

Installation

Architecture

选择server-agent(active)模式进行zabbix的搭建。

为了减轻server端的压力，保持良好的用户体验，所有zabbix agent我们均使用主动(active)模式

安装说明

zabbix agent, server, proxy均采用 3.2.6 版本

preparations

  • docker has installed in host
  • docker image(官方下载)
    • zabbix-java-gateway download link:https://hub.docker.com/r/zabbix/zabbix-java-gateway/
    • zabbix-web-nginx-mysql download link: https://hub.docker.com/r/zabbix/zabbix-web-nginx-mysql/
    • mysql 5.7

  • docker image(build by yourself) 在启动zabbix之前，为了在zabbix server中使用curl指令，需要自行build zabbix-server-mysql image.

    • 进入zabbix docker github project https://github.com/zabbix/zabbix-docker,选择tag为3.2.6版本进行download或者clone.
    • 在该项目下，进入server-mysql/ubuntu,修改DockerFile如下:
         34     apt-get ${APT_FLAGS_COMMON} update && \
     35     apt-get ${APT_FLAGS_PERSISTANT} install \

    即在47行添加安装curl.

    • 在该目录下，执行脚本，build image:
       ./build.sh <your-image-version>
    • 导出image(optional)
      docker save zabbix-mysql-server.tar zabbix-mysql-server:ubuntu-<your-image-version>
    • 导入image(optional)
      docker load < zabbix-mysql-server.tar

安装流程

  • Start empty MySQL server instance
docker run --name mysql-server -t \
      -e MYSQL_DATABASE="zabbix" \
      -e MYSQL_USER="zabbix" \
      -e MYSQL_PASSWORD="zabbix_pwd" \
      -e MYSQL_ROOT_PASSWORD="root_pwd" \
      -d mysql:5.7
  • Start Zabbix Java gateway instance
docker run --name zabbix-java-gateway -t \
      -d zabbix/zabbix-java-gateway:latest
  • Start Zabbix server instance and link the instance with created MySQL server instance
docker run --name zabbix-server-mysql -t \
      -e DB_SERVER_HOST="mysql-server" \
      -e MYSQL_DATABASE="zabbix" \
      -e MYSQL_USER="zabbix" \
      -e MYSQL_PASSWORD="zabbix_pwd" \
      -e MYSQL_ROOT_PASSWORD="root_pwd" \
      -e ZBX_JAVAGATEWAY="zabbix-java-gateway" \
      --link mysql-server:mysql \
      --link zabbix-java-gateway:zabbix-java-gateway \
      -p 10051:10051 \
      -d zabbix/zabbix-server-mysql:latest

docker run --name zabbix-server-mysql -t \
      -e DB_SERVER_HOST="mysql-server" \
      -e MYSQL_DATABASE="zabbix" \
      -e MYSQL_USER="zabbix" \

      -e MYSQL_ROOT_PASSWORD="root_pwd" \
      -e ZBX_JAVAGATEWAY="zabbix-java-gateway" \
      -p 10051:10051 \
      -d zabbix/zabbix-server-mysql
  • Start Zabbix web interface and link the instance with created MySQL server and Zabbix server instances
docker run --name zabbix-web-nginx-mysql -t \
      -e DB_SERVER_HOST="mysql-server" \
      -e MYSQL_DATABASE="zabbix" \
      -e MYSQL_USER="zabbix" \
      -e MYSQL_PASSWORD="zabbix_pwd" \
      -e MYSQL_ROOT_PASSWORD="root_pwd" \
      --link mysql-server:mysql \
      --link zabbix-server-mysql-curl:zabbix-server \
      -p 8080:80 \
      -d zabbix/zabbix-web-nginx-mysql:latest

  wget http://repo.zabbix.com/zabbix/3.2/ubuntu/pool/main/z/zabbix-release/zabbix-release_3.2-1+trusty_all.deb
  dpkg -i zabbix-release_3.2-1+trusty_all.deb
  apt-get update
  apt-get install zabbix-agent

The repo release can be found at http://repo.zabbix.com/zabbix/3.2/ubuntu/pool/main/z/zabbix-release/ for Zabbix 3.2

配置说明

按以上步骤

item

the config workflow is as below: create host group --> create host --> create item(to collect data) --> create trigger(make a fire on collected data) --> create action(trigged by trigger) --> config alert.

the major work is define variable items.

for our projects,the basic item will do the following job:

  • monitoring host status
  • monitoring services status
  • monitoring host io usage
  • monitoring host disk usage
  • monitoring host memory usage
  • monitoring host cpu usage
  • monitoring log
  • monitoring container status
  • monitoring response time
  • monitoring host workload

自定义监控(主动模式下)

在某些情况下，zabbix中内置的监控项不能满足自己的监控需求，需要自己编写脚本来完成某种监控数据的收集，这时候需要配置自定义监控项，即UserParameter.

  • 编辑配置文件zabbix_agentd.conf

    zabbix agent配置文件的默认路径为/etc/zabbix/,进入该目录后修改配置文件，以我们目前监控api延迟为例，添加如下内容

    UserParameter=web.response[*], bash /etc/zabbix/scripts/web_application.sh $1 $2

    Timeout=30 #设置执行检查项目过期时间为30秒

    其中web.response即为配置zabbix时填入的item,而[]代表输入的参数，`UserParameter=web.response[], bash /etc/zabbix/scripts/web_application.sh $1 $2`则是实际run在宿主机上的监控指令。

    然后保存并退出

    note:监控脚本自己要先写好并且放在正确的path下，上例中可见我的监控脚本就放在/etc/zabbix/scripts/下

  • test

    执行以下指令

    zabbix_agentd -t web.response[login,time]

    如果正常的话，会看到如下内容

    web.response[login,time]                      [t|0.133]

  • 重启zabbix agent

    执行以下指令

    service zabbix-agent restart
