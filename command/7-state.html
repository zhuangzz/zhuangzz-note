npm使用方法

NPM 是随同 Node 一起安装的包管里工具。

npm install --save:

  • 允许用户从NPM服务器下载别人编写的第三方包到本地使用。
  • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

npm install --save-dev:

  • 会把msbuild包安装到node_modules目录中

  • 会在package.json的devDependencies属性下添加msbuild

  • 之后运行npm install命令时，会自动安装msbuild到node_modules目录中

  • 之后运行npm install --production或者注明NODE_ENV变量值为production时，不会自动安装msbuild到node_modules目录中

运行时需要用到的包（工具包）使用--save，否则（非工具类包）使用--save-dev。

npm init 把项目编程npm项目

npm install jquery --save下载非工具类的包

npm install jquery@1 --save安装1..的最高版本

npm uninstall jquery移除

npm install webpack --save-dev下载工具类的包

npm uninstall webpack删除webpack工具包

npm i装所有包

.gitignore 建在仓库下（而不是包里）,里面写node_modules

npm ls 装的包

npm ls -g 全局装的包

node三模块(包)：核心模块、自定义模块、第三方（node运行js代码）

package.json 里main是入口文件

package.json 里script是脚本

分支

使用gh-pages托管静态页面

github上的仓库内容，只有查看gh-pages分支的网址，才能查看仓库页面，所以要创建分支

  • git branch 查看分支

  • git status(创建分支的前提是主分支上所有内容提交完毕)

  • git add .

  • git commit -m"ss"

  • git status

  • git push

  • git branch gh-pages创建新分支

  • git branch checkout gh-pages切换到这分支

git checkout -b gh-pages 创建并换到这分支

git status

分支的更新及合并

更新主分支在主分支山做修改 git merge gh-pages , git status , git push

git pull origin master 拉取主分支上的更新

删除分支git branch -d [yourbranch]

git push origin :gh-pages删除远端分支
