  • 原型 构造函数名.prototype

每创建一个构造函数，都会生成一个原型对象(CreatObj.prototype)。通过 构造函数.prototype 去获取或修改

原型作用：存储或设置 构造函数的公有属性或方法

CreatObj.prototype.say=function(){ console.log(22) }

构造函数创建的时候，先添加构造函数内部的属性和方法，然后添加原型内部的属性和方法，如果重名，使用构造函数内部的属性和方法

构造函数的原型只有一个，只要修改了，所有实例化对象的公有方法或属性都会改变

proto每当实例化对象被创建时，都会自动生成一个proto属性，这个属性指向该构造函数的原型

console.log(obj.__proto__)
obj.__proto__.name="小红"

constructor 每个原型对象都有一个constructor属性，这个属性指向该构造函数

console.log(CreateObj.prototype.constructor)

原型链 继承 对象冒充

继承 将一个用构造函数创建的实例化对象 赋值给另一个构造函数的原型，那么用后面的构造函数创建的实例化对象 继承前面的实例化对象的属性和方法

当访问一个实例化对象的属性时，首先去创建这个对象的构造函数里面找，然后去这个构造函数的原型里面找，最后去继承的实例化对象里面找（规则和前面一样）（实例化对象被包含在构造函数的原型里面）


                function A(){}

　　　　function B(a){
　　　　　　this.a = a;
　　　　}

　　　　function C(a){
　　　　　　if(a){
　　　　　　　　this.a = a;
　　　　　　}
　　　　}

　　　　A.prototype.a = 1;
　　　　B.prototype.a = 1;
　　　　C.prototype.a = 1;
　　　　console.log(new A().a);    // 1   
　　　　console.log(new B().a);    // undefined  
　　　　console.log(new C().a);   // 1

        function Foo() {
            getName = function () { console.log(1)};

        getName(); //1(如果这个和上一句换位置，结果就是4)    
        new Foo.getName(); //

对象冒充 call apply

当某个对象执行某个方法时，可以用call或apply,替换方法中this的指向

obj.say.call(obj1,参数) 把say方法中的this替换成obj1对象

obj.say.apply(obj1,[参数])

call和 apply区别，后者传参数时只能传两个，第一个是要冒充的对象，第二个是数组（即call一个一个传参数，apply是传一个数组，数组里是所有要传的参数）

    this.name="ani"
    this.say=function(){
    console.log(112)
}
}

    this.name="Bnn"
}
